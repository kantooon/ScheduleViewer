ScheduleViewer Help



Basic usage:
Import schedules in *.conf format:

Go to File -> Import and choose the path (directory) where to look for conf files. Leave empty the Depth level field to look only in this directory. Enter a number in this field to make the import tool search all directories below the selected one, at the level you give via this number. For example, if you enter 1, the import tool will look in all subdirectories of the specified path, but not in the specified directory, or in any subdirectories nested at a depth greater than 1.
This is usefull if you have cloned the AI schedule repository, and you want to scan all directories of airlines it includes, without scanning the root directory or the subdirectories which contain separate files for each airline resulting from a common PDF timetable.


Export schedules from the database in *.conf format

Go to File -> Export and choose the path (directory where the conf files will be written), then check the Separate airlines to write separate files for each airline (default option).
Next choose which flight schedules will be exported (all flights in the table being displayed by filters, only flights selected by you, or the whole database of flight schedules.


Using the Filter query tool window

The Filters help you display flight schedules that fulfill your criteria.
In the filter fields, enter airport, time, type or airline and click Show. The airport names are standard ICAO, the airline names are standard ICAO, and the aircraft type are standard IATA. Times are format HH:MM. The day is a number from 0 to 6, with 0 being Sunday. To clear the table view and the filter form, click Clear.

The Empty database button truncates the Flights database table, leaving an empty slate. Use with care.

The Edit flight and Delete flights buttons are obvious, but you can select multiple flights at the same time with CTRL and SHIFT to delete Double click a table field to edit it, when you are finished click Edit.


Getting more help and reporting bugs

The support forum is located at the Flightgear AI traffic forum.
If you have encountered a bug, please bear in mind that this project is developed with very limited resources. If you report the bug, chances are good that it will be solved. When reporting bugs, please provide a complete description of the problem, together with the debug and error logs available in the /logs directory.
